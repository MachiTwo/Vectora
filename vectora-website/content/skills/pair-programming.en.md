The Pair Programming skill transforms Vectora into an intelligent code validator that works in real-time alongside the primary model. While Claude or Gemini writes code, Vectora assists, validates against the existing codebase, and suggests improvements, creating a true collaborative partnership between two agents.

By acting as a navigator and reviewer, Vectora ensures that the code generated by the AI adheres to project standards and best practices.

## Concept: Vectora as a Code Reviewer

Vectora monitors the code generation process and provides immediate feedback based on the project's historical context and defined rules.

```text
Claude: "I need to authenticate users via JWT."
  ↓
Claude generates authentication code.
  ↓
Vectora (Pair Programming Skill):
   Validates Go syntax.
   Detects 3 similar patterns in internal/auth/.
   Warning: Missing Guardian.ValidateAccess() call.
   Suggestion: Use the existing function in internal/auth/guardian.go.
   Refactor: Extract validateToken() as a reusable function.
   Result: Code approved with suggestions.
```

## Activation Methods

You can utilize the Pair Programming skill either through explicit tool calls or automated background monitoring.

### Explicit MCP Tool

The primary model can invoke the skill directly by providing the code snippet and context.

```text
Claude: /pair_programming {language} {code_snippet}
```

### Automatic Integration

Vectora can be configured to automatically trigger the skill whenever code generation or refactoring is detected.

```yaml
skills:
  pair_programming:
    enabled: true
    auto_mode: true
    validation_level: "strict"
```

## Validation Dimensions

The skill performs a multi-layered analysis of the proposed code to ensure high quality and consistency.

### 1. Pattern Compliance

Vectora compares the new code against established patterns in the repository to prevent duplication and ensure consistency in naming and structure.

### 2. Layered Architecture Validation

Ensures that the code respects the project's dependency hierarchy. For example, it checks that core logic does not accidentally depend on the API or server layers.

### 3. Security Checklist

Every function touching sensitive data is verified for input sanitization, secure logging (no credentials in logs), and proper access control via the Guardian module.

### 4. Duplicate Code Detection

Identifies if similar logic already exists elsewhere in the project, suggesting reuse of existing functions rather than creating redundant implementations.

### 5. Dependency Review

Analyzes new external dependencies and suggests native or already existing alternatives within the project to keep the dependency graph clean.

## Interaction Workflow

When a code snippet is submitted, the skill returns a comprehensive analysis report.

1. **Syntax Check**: Basic validation of code correctness for the specified language.
2. **Similar Patterns**: List of existing files and lines that implement similar logic.
3. **Architecture Check**: Validation of layer isolation and dependency rules.
4. **Security Warnings**: Identification of potential vulnerabilities or missing validations.
5. **Refactoring Suggestions**: Actionable advice on how to improve the code structure.

## Project Configuration

The Pair Programming skill can be customized in the `vectora.config.yaml` file to match your project's specific requirements.

```yaml
# vectora.config.yaml
skills:
  pair_programming:
    enabled: true
    checks:
      syntax: true
      architecture: true
      security: true
      duplicates: true
    refactoring:
      suggest_extractions: true
    similar_patterns:
      threshold: 0.85
```
