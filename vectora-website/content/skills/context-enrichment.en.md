The **Context Enrichment** skill allows Vectora to elevate code understanding through the injection of semantic metadata and deep codebase relationships.

Unlike simple text search, this skill maps how components interact and provides the primary AI model with the knowledge needed to make informed decisions about architecture and logic.

## Capabilities

Context Enrichment focuses on three fundamental pillars:

1. **Semantic Relationships**: Identifies how functions and types relate throughout the project, even in distant files.
2. **Decision History**: Retrieves context from past changes and established patterns in the codebase.
3. **Flow Mapping**: Helps understand the execution path of a specific feature.

## How It Works

When activated, the skill traverses the knowledge graph generated by the Context Engine to find connections that would not be visible in a pure vector search.

- **AST Analysis**: Uses Tree-sitter to understand the syntactic structure.
- **Metadata Injection**: Adds information about types, interfaces, and implementations to the prompt.
- **Smart Compaction**: Ensures that only the most relevant context is injected, respecting the token window.

## Activation

You can activate this skill manually through the MCP command or via CLI flags:

- **MCP Tool**: `/context_enrichment`
- **Automatic Use**: Vectora triggers this skill when it detects queries involving multiple modules or complex architectural patterns.

## Usage Example

```bash
# Requests enriched analysis of a specific module
vectora search "How does the authentication system handle refresh tokens?" --enrich
```

## Benefits

- **Hallucination Reduction**: By providing real context, the AI model speculates less about internal workings.
- **Accurate Responses**: Code suggestions that follow exactly the existing patterns in your project.
- **Accelerated Discovery**: Ideal for developers joining new projects who need to understand the "big picture."
