# Product Marketing Manager Routine

## Feature Communication

**For each feature launch**:
1. **Positioning**: How does this differentiate?
2. **Messaging**: Key benefits in 1-2 sentences
3. **Audience**: Who cares most?
4. **Channels**: Twitter, blog, community
5. **Timing**: Coordinated with dev launch

**Example**:
```
Feature: AuthMiddleware (Issue #001)
Positioning: Production-ready authentication
Message: "Vectora Cloud API now enforces security with JWT + API Key auth"
Audience: Enterprise DevOps, security teams
Channels: Twitter, GitHub release notes
```

---

## Content Strategy

**Messaging Hierarchy**:
1. **Hero**: "Self-hosted AI agent platform"
2. **Sub-message**: "Enterprise control + vector search"
3. **Features**: Auth, storage, logging, etc.
4. **Benefits**: Security, cost, customization

**Content Types**:
- Launch announcements (Twitter, blog)
- Technical deep-dives (blog posts)
- Demo videos (YouTube, Twitter)
- Competitive analysis (internal)
