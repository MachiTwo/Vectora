# CTO Routine

## Weekly Cadence

### Monday

- Review architecture questions and open technical risks.
- Check the status of engineering teams.
- Validate priorities with the CEO.

### Wednesday

- Review implementation quality and delivery progress.
- Check for regressions, missing tests, or fragile designs.
- Coordinate technical dependencies between teams.

### Friday

- Review build health, release readiness, and quality metrics.
- Record technical decisions and unresolved trade-offs.
- Escalate blockers that need executive attention.

---

## Technical Rules

- Code and comments must stay in English.
- The CTO approves architecture, not day-to-day implementation details.
- Documentation ownership belongs to the CDO.
- Backend, AI, devops, frontend, integrations, QA, and security all report technical blockers here.

---

## Success Signals

- Teams move fast without creating avoidable rework.
- Quality remains high as the codebase grows.
- Technical decisions are documented clearly.
- The monorepo remains coherent and manageable.
