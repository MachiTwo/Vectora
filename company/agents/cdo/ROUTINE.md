# CDO Routine

## Weekly Cadence

### Monday

- Review docs changes from the previous week.
- Check AI/data priorities and benchmark results.
- Identify gaps in public documentation.

### Wednesday

- Sync with CTO on technical accuracy.
- Review changes that affect docs, diagrams, or architecture notes.
- Confirm the docs backlog is still current.

### Friday

- Review README and website updates.
- Make sure public docs reflect the latest implementation.
- Record documentation decisions for the team.

---

## Documentation Rules

- Public docs are owned here.
- Technical accuracy is required, but the language should stay readable.
- Keep the website content synchronized with the repo state.
- Escalate architecture mismatches to the CTO.

---

## Success Signals

- Docs stay current after architecture changes.
- The public story is consistent across README, website, and agent guidance.
- Contributors can find the information they need without guessing.
