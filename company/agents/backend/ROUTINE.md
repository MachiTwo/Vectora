# Backend Engineer Routine

## Weekly Cadence

### Monday

- Review assigned backend work.
- Check API, storage, and auth priorities.
- Confirm test expectations with QA.

### Wednesday

- Implement and refine backend changes.
- Keep the codebase small and testable.
- Escalate architecture questions to the CTO.

### Friday

- Finish the week with tests and cleanup.
- Confirm the backend is ready for review.
- Note anything that should become docs work for the CDO.

---

## Success Signals

- Backend changes are simple and reliable.
- Tests cover the important flows.
- Code remains readable and in English.
